Phase 4 — Intelligence Upgrade

Smarter,
Deeper

The pipeline is live — now we make it authoritative. Trending detection groups multi-source stories, Writer v4 produces 1,500-word long-form journalism, and a dedicated SEO Agent automates every post's metadata before it touches Ghost.

Current Limitations
✕
500-word commodity rewrites
Single-source, surface-level articles. No depth, no authority, no differentiation from the original RSS post.
✕
No trending detection
Every story treated equally — a major industry event and a minor product update get the same priority and treatment.
✕
No automated SEO
Ghost receives raw content with no seo_title, meta_description, focus_keyword, or slug optimisation. All left blank.
✕
No regulatory intelligence
High-impact legislation (Acts, Mandates, Compliance deadlines) processed the same as general news — no special analysis.
New Pipeline Architecture
WF1
Scout v4
Upgrade
Jaccard clustering groups stories by topic. Trending flag fires when 3+ sources cover the same event within 2 hours. Triggers Writer immediately.
WF2
Writer v4
Upgrade
1,000–1,500 word long-form articles. Six-section journalism structure. Trending articles jump the queue and are prioritised.
WF5
SEO Agent
New
GPT-4.1-mini generates seo_title, meta_description, focus_keyword, and slug per article. Cost: ~$0.00007 each.
WF3
Publisher v3
Upgrade
Waits for seo_ready = TRUE before picking up an article. Passes all SEO fields directly to Ghost — og_title, meta, slug.
What Changes in Each Agent
WF1 · Scout Agent
Scout v4
Building
→
Jaccard clustering — tokenises article titles and groups any two with similarity > 0.25 into the same cluster
→
Trending detection — if a cluster has 3+ sources published within 2 hours, it's flagged is_trending = TRUE with a source_count
→
Immediate Writer trigger — trending clusters bypass the 30-min queue and fire a Writer webhook instantly
→
New queue columns — is_trending, source_count, cluster_id added via migration
WF2 · Writer Agent
Writer v4
Building
→
1,000–1,500 words — max_tokens raised from 1,200 → 2,500. Turns surface rewrites into authoritative analysis
→
Six-section structure — Hook · What Happened · Why It Matters (cost / timeline / supply-chain impact) · Industry Context · What's Next · Bottom Line
→
Trending priority — query adds ORDER BY is_trending DESC so breaking stories are written first
→
Cost per article — ~$0.02–0.025 (up from ~$0.008 for 500-word articles)
WF5 · New Workflow
SEO Agent
New WF5
→
Polls published table — picks up every article where seo_ready = FALSE on a 30-min cron
→
GPT-4.1-mini generates: seo_title, meta_description, focus_keyword, slug
→
Cost — ~$0.00007 per article (10× cheaper than Writer). SEO layer costs under $0.01/day at current volume
→
Sets seo_ready = TRUE — Publisher v3 only picks up articles after this flag is set, ensuring nothing reaches Ghost without optimised meta
WF3 · Publisher Agent
Publisher v3
Building
→
SEO gate — query filter adds seo_ready=eq.true so no article publishes without a slug and meta description
→
Ghost SEO payload — passes og_title, meta_description, slug, custom_excerpt directly to Ghost Admin API
→
Newsletter — using Ghost's built-in newsletter distribution. No external Beehiiv dependency. Curator Agent (WF4) deactivated.
