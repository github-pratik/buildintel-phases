25 RSS Feeds — Parallel Fetch
All feeds fetched simultaneously. Tier 1: Construction Dive, AEC Magazine, MIT Tech Review, VentureBeat AI, AI Business, Smart Cities Dive, The Construction Index, Global Construction Review. Tier 2: ArchDaily, Mfg Dive, Dezeen + 9 more. Tier 3: TechCrunch, Robot Report + 3 more.
Filter & Score
Each article scored 0–20+. High keywords (BIM, AI, digital twin, contech) → +3 each. Mid keywords (construction, infrastructure) → +1.5 each. Recency: <6h → +4, <24h → +2. Source tier bonus: T1 → +3, T2 → +1.5, T3 → +0.5. Threshold ≥ 4. Output: top 30 by score. In-memory URL dedup across feeds.
Supabase Dedup Check
Single batch IN query against published.source_url. Drops articles already in the database. Fail-open: if Supabase unreachable, all articles pass through.
Scrape Full Article
HTTP GET each new article URL. Extracts body from <article> / <main> / <body> tags. Strips scripts, nav, footer, ads. Pulls og:image and author byline. Up to 4,000 chars of real content. Falls back to RSS snippet if scrape fails (paywall, bot block).

Webhook Trigger
Receives POST from Scout Agent at /webhook/writer-agent
Parse Article JSON
Validates required fields: title, link, fullContent. Extracts source, category, score, image, author.
Groq LLaMA 3.3 70B — Article Generation
System prompt: Senior journalist at IndustrialBriefs, covering ACEM + AI sectors, Semafor format (What Happened / Why It Matters / What's Next). User prompt: title + source + fullContent (4,000 chars). Returns strict JSON: { title, body, summary, tags }.
Valid Article? (Filter node)
Checks JSON parsed correctly and body length > 200 chars. Invalid articles are dropped silently.
Save to Supabase
INSERT into published table: source_url, title, summary, category, source, published_at. Sets ghost_url and ghost_post_id to null (filled by WF3).
Ready for Publisher (NoOp)
Handoff point. WF3 Publisher Agent will connect here when Ghost CMS is set up.
