AI Concrete Monitoring Reduces Construction Waste by 30% Across Major Infrastructure Sites

ConcreteAI's sensor-driven platform flags curing anomalies before material failure — cutting rework costs on bridge decks, tunnels, and high-rise foundations.

IndustrialBriefs Newsroom · Mar 14, 2026 · 4 min read
ConcreteAI sensor array installed at a bridge deck pour, Chicago, IL. Photo: ConcreteAI

A new AI monitoring system from ConcreteAI is showing measurable results on large-scale infrastructure builds, reducing material waste by up to 30% by catching curing anomalies before they escalate into full rework events.

The platform uses embedded IoT sensors combined with a machine learning model trained on 4 million concrete pour data points. When temperature, humidity, or chemical composition deviates from expected cure curves, site managers receive a Slack or SMS alert within 90 seconds.

How the system works on a live pour

Sensors are embedded at 12-inch intervals across the pour zone, transmitting readings every 30 seconds. The AI model compares incoming data against cure curve baselines for the specific mix design, ambient conditions, and pour depth in real time.

"We caught a cold joint forming at 3am on the Lakeshore span — something that would have required a full deck section removal if we'd found it at inspection."
— Mark Stellan, Project Manager, Walsh Construction

The green background bug — cause & fix
Your Ghost theme has a tag accent colour variable that's leaking into the page body background. This is a one-line CSS fix — add it to your theme's screen.css
THE PROBLEM — Ghost tag colour bleeding into article background
/* Ghost sets this CSS variable based on the post's primary tag */
/* When the tag is "Construction", Ghost sets: */
--ghost-accent-color: #16a34a;  /* your engineering green */

/* And then the theme's article template does something like: */
.post-template {
  background-color: var(--ghost-accent-color); /* ← THIS IS THE BUG */
}
THE FIX — add these rules to screen.css
/* ── GREEN BACKGROUND BUG FIX ──────────────────────── */
/* Force white background on ALL article/post pages */
.post-template,
.post-full,
.article-page,
.post-content-wrap,
body.post-template {
  background-color: #ffffff !important;
  color: #1c1c1c !important;
}

/* Scope accent colour ONLY to tags — never backgrounds */
.tag,
.post-tag,
.article-tag {
  background-color: #fafaf8;  /* cream, not the accent colour */
  color: inherit;             /* accent colour applied via JS class */
}

/* Sector tag classes — colour ONLY on text + left border */
.tag-construction { color: #e96d2e; border-left: 2px solid #e96d2e; }
.tag-engineering  { color: #16a34a; border-left: 2px solid #16a34a; }
.tag-architecture { color: #3b82f6; border-left: 2px solid #3b82f6; }
.tag-manufacturing{ color: #d97706; border-left: 2px solid #d97706; }
.tag-ai           { color: #7c3aed; border-left: 2px solid #7c3aed; }
Why this happens: Ghost uses a CSS custom property --ghost-accent-color that it sets based on the post's primary tag colour in Ghost Admin. Some themes accidentally apply this variable to the page background instead of scoping it only to UI accent elements like buttons and tags. The fix above overrides it with an explicit white background on article templates.
